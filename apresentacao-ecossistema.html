O go-live de Maio é inegociável. As lanes paralelas são oportunísticas — só rodam se houver capacidade sobrando.

Marcos

O que precisa estar pronto a cada mês.

MAR
Fundação
○ Data Hub operando — Postgres + API Gateway + Auth JWT
○ Entidades base no CRUD — Empresa, Regional, Cidade, Consultor, Equipe, Tabela de Preço
○ Entidades de cadastro — Associado, Veículo, Plano, Serviço, Beneficiário, Boleto
○ CRM V1 funcional — Funil de vendas completo
○ Tabelas de preço — Regionais, planos, serviços cadastráveis
○ Perfis e usuários — Vendas e Administrativo com permissões
○ ETL mapeado — Campos SGA → núcleo, estratégia de sync definida
ABR
Integração & Homologação
○ CRM integrado ao Data Hub
○ Boletos integrados ao Data Hub — geração, listagem, envio
○ ERP com telas integradas às APIs reais
○ ETL sync contínuo rodando em staging
○ Sistemas leitores migrados para BD próprio
○ Vistoria integrada ao CRM e Data Hub
○ Homologação comercial — CRM validado
○ Homologação financeira — Boletos validados
○ Fluxo de dados com terceiros mapeado
○ Plano B para terceiros sem API definido
○ Checklist de go-live pronto
MAI
Go-live
○ Carga final + reconciliação financeira
○ SGA em somente-leitura
○ Go-live CRM + Vistoria — Power CRM e Visto desligados
○ Go-live Boletos — módulo SGA desligado
○ Go-live ERP — telas administrativas substituídas
○ Integrações com fornecedores — Regalias e ClubGas via Data Hub
○ Estabilização — suporte pós go-live, plantão

Condições de Go-live

O go-live só acontece se TODAS estas condições forem atendidas.

Núcleo com todos os CRUDs estáveis e auth funcionando
CRM validado pelo time comercial
Vistoria integrada ao CRM e Data Hub
Boletos homologados com integração SICOOB
ETL sync contínuo estável (>95% aderência)
Sistemas leitores no BD próprio
Módulo Admin (ERP) com fluxos críticos
Fornecedores (Regalias, ClubGas) via Data Hub
Nenhuma demanda extra entrou no escopo

Riscos & Bloqueadores

⚠
Instabilidade do SGA
Operação para quando SGA cai. Mitiga com ETL sync contínuo — migrar leituras para BD próprio o mais cedo possível.
⚠
Data Hub atrasar
Bloqueia todos os fluxos. Proteger o time de interrupções em Março.
⚠
Integração bancária SICOOB
Boletos dependem da integração funcionar. Priorizar integração com API SICOOB.
⚠
ETL com dados inconsistentes
Migração falha. Validação contínua em staging.
⚠
Escopo crescer
Go-live de Maio inviável. Congelar escopo — usar framework de negociação com a diretoria.
